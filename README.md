# KanTan Marp

KanTanMarkdown に影響を受けた、**ブラウザ上で Marp スライドを編集・プレビュー・プレゼンできる**ローカル実行向けプロジェクトです。  
クラウド依存を避け、最終的には **単一 HTML ファイルとして配布・実行できる**ことを目指します。

参考: https://tatesuke.github.io/KanTanMarkdown/

## 目標

- ブラウザだけで Marp 形式の編集ができる
- 即時プレビューでスライド表示を確認できる
- プレゼンテーションモードで全画面投影できる
- HTMLファイルを開いた直後はプレゼンテーションモードをデフォルト表示にする
- ユーザーがCSSを直接編集して見た目を調整できる
- コードブロックのシンタックスハイライトに対応する
- 画像をHTMLファイルに添付して配布できる
- オフラインで動作し、クラウドサービスを不要にする
- 単一の HTML ファイルとして持ち運べる構成にする

## 成果物バージョン

最終成果物として、以下の 2 バージョンを提供します。

- 軽量版（CDN依存）: 外部 CDN から必要アセットを読み込み、HTML 本体を軽量化する
- セキュリティ重視版（完全オフライン）: 外部 CDN を一切読み込まず、必要アセットをすべて同梱する

想定出力例:

- `dist/kantan-marp-lite.html`
- `dist/kantan-marp-secure.html`

## 確定仕様

### レンダリング・ハイライト

- Marp レンダリングは `@marp-team/marp-core` を第一候補として採用する
- コードシンタックスハイライトは主要言語をサポートする
- 最低対応言語: `HTML / CSS / JavaScript / TypeScript / Python / PHP`

### 画面遷移・操作

- HTML を開いた直後はプレゼンテーションモードで開始する
- 起動直後は通常表示 UI を出さず、スライド表示を優先する
- `Esc` キーで編集モードと表示モードを切り替える

### CSS 編集

- ユーザーは CSS を自由入力で直接編集できる
- CSS はプレゼンの見た目に即時反映されることを目標とする

### 添付ファイル

- 添付対象は画像のみとする（`png / jpg / webp`）
- 画像添付は HTML 内へ `data:` URL として埋め込む方式を採用する
- `Ctrl+V` によるクリップボード貼り付けフォームを提供する
- PDF 添付機能は現時点では対象外とする

### 保存仕様

- 自動保存・上書き保存は実装しない
- 保存ボタン押下時に単一 HTML ファイルをダウンロードする
- ダウンロードされる HTML には、その時点の Markdown / CSS / 添付画像を同梱する

### CDN とセキュリティ

- 軽量版の CDN は信頼性の高い配信元を利用する（`jsDelivr` を想定）
- セキュリティ重視版は外部通信ゼロで動作すること
- セキュリティ重視版は厳格な CSP を適用すること
- CSP では `data:`（埋め込み画像）と `blob:`（ダウンロード処理）の利用を許可する

### 対応環境

- 対応ブラウザは Chrome を優先する

## 開発方針

- 依存を最小限にし、構成をシンプルに保つ
- ブラウザ標準 API を優先して採用する
- 「インストール不要・すぐ使える」体験を最優先する
